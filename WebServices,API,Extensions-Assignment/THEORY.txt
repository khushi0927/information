2] API WITH HEADER :

1. OBJECTIVE :

  1) : Learn about the significance of headers in API requests and responses.
  => 
	---------------------------------------|| SIGNIFICATION ||---------------------------------------------

  -> Content Negotiation
  -> Authentication & Authorization
  -> Security
  -> Caching Control
  -> Session & State Management
  -> Rate Limiting & Usage Control
  -> Error Handling & Status Information
  -> Cross-Origin Resource Sharing (CORS)
  -> Custom Metadata

2. QUESTION :

  1)  What are HTTP headers ,and how do they facilitate communication between client and server?
  => 
	------------------------|| HTTP HEADER ||-------------------------------------

  -> HTTP headers are key–value pairs sent along with an HTTP request or response.
  -> They carry metadata (information about the data) rather than the actual data itself.
 
	--------------------------|| HOW HTTP HEADER FACILITATES COMMUNICATION BETWEEN CLIENT AND SERVER ||-----------------------------------

  -> Describe the Data Format
  -> Enable Authentication & Authorization
  -> Ensure Secure Communication
  -> Control Caching Behavior
  -> Maintain Sessions & State
  -> Support Error Handling


  2)  Describe how to set custom headers in an API request.
  => 
	-------------------------|| CUSTOM HEADER ||-----------------------------

  -> Custom headers are user-defined HTTP headers added to an API request to pass extra information such as authentication tokens, app version, request IDs, or client metadata.
  -> Custom headers are non-standard headers created by developers, usually prefixed with X- or meaningful names.

4]  SOAP AND REST APIs :

1. OBJECTIVE :

  1) Differentiate between SOAP and RESTAPI architectures.
  => 

| Feature                | SOAP (Simple Object Access Protocol) | REST (Representational State Transfer) |
| ---------------------- | ------------------------------------ | -------------------------------------- |
|   Type                 | Protocol                             | Architectural style                    |
|   Data Format          | XML only                             | JSON (most common), XML, HTML, text    |
|   Communication        | Uses XML-based messaging             | Uses standard HTTP methods             |
|   Transport Protocol   | HTTP, HTTPS, SMTP, TCP               | Primarily HTTP/HTTPS                   |
|   State                | Can be stateful or stateless         | Stateless                              |
|   Performance          | Slower due to heavy XML processing   | Faster and lightweight                 |
|   Security             | WS-Security (advanced, built-in)     | Relies on HTTPS, OAuth, JWT            |
|   Ease of Use          | Complex and strict                   | Simple and flexible                    |
|   Caching              | Not easily cacheable                 | Supports HTTP caching                  |
|   Error Handling       | Uses SOAP Faults                     | Uses HTTP status codes                 |
|   Scalability          | Less scalable                        | Highly scalable                        |
|   Use Case             | Enterprise, banking, payment systems | Web & mobile applications              |



2. QUESTIONS :
 
  1)  What are the key characteristics of SOAPAPIs.
  => 
	-----------------------|| KEY CHARACTERISTIC ||----------------------------------------

  -> Uses PHP SOAP Extension
  -> WSDL-Based Communication
  -> XML-Only Data Format
  -> Strongly Typed Parameters
  -> Built-in Error Handling
  -> Stateful or Stateless


  2) Describe the principles of RESTfulAPI design.
  => 
	-----------------------------|| PRINCIPLES ||-----------------------------------
 
  -> Client–Server Separation
  -> Stateless Requests
  -> Resource-Based URLs
  -> Use HTTP Methods Properly
  -> Proper HTTP Status Codes
  -> Input Validation & Sanitization
